# 订单状态变动日志功能需求文档

## 1. 需求概述

### 1.1 需求背景
为满足企业内部审计和追溯需求，需要对采购订单和销售订单的状态变动进行记录，包括操作人、操作时间、变动前状态、变动后状态等信息。

### 1.2 需求目标
- 记录采购订单状态变动日志
- 记录销售订单状态变动日志
- 记录操作人信息
- 记录变动时间和变动详情

## 2. 功能需求

### 2.1 日志记录范围

| 订单类型 | 状态变动场景 | 说明 |
|---------|------------|------|
| 采购订单 | 待审核 → 已审核 | 审核操作 |
| 采购订单 | 已审核 → 已入库 | 入库操作 |
| 采购订单 | 待审核 → 已取消 | 取消操作 |
| 销售订单 | 待审核 → 已审核 | 审核操作 |
| 销售订单 | 已审核 → 已出库 | 出库操作 |
| 销售订单 | 待审核 → 已取消 | 取消操作 |

### 2.2 日志记录内容

| 字段 | 说明 | 类型 |
|------|------|------|
| 订单ID | 关联的订单ID | BIGINT |
| 订单类型 | 1=采购订单，2=销售订单 | INT |
| 变动前状态 | 变动前的订单状态 | INT |
| 变动后状态 | 变动后的订单状态 | INT |
| 操作人ID | 执行操作的用户ID | BIGINT |
| 操作人姓名 | 执行操作的用户姓名 | VARCHAR |
| 操作时间 | 操作发生时间 | DATETIME |
| 备注 | 操作备注（可选） | VARCHAR |

## 3. 非功能需求

### 3.1 性能要求
- 日志记录操作不影响主业务流程性能
- 日志查询响应时间 < 1秒

### 3.2 数据安全
- 日志记录不可修改和删除（只读）
- 操作人信息必须记录真实用户

## 4. 接口需求

### 4.1 查询日志列表接口
- **路径**: `/order-log/list`
- **方法**: GET
- **参数**:
  - orderId: 订单ID（可选）
  - orderType: 订单类型（可选）
  - operatorId: 操作人ID（可选）
  - startTime: 开始时间（可选）
  - endTime: 结束时间（可选）

### 4.2 查询日志详情接口
- **路径**: `/order-log/get/:id`
- **方法**: GET

## 5. 数据模型设计

### 5.1 订单日志表（order_logs）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT UNSIGNED | PRIMARY KEY AUTO_INCREMENT | 主键 |
| order_id | BIGINT UNSIGNED | NOT NULL, INDEX | 订单ID |
| order_type | INT | NOT NULL, DEFAULT 1 | 订单类型：1=采购，2=销售 |
| before_status | INT | NOT NULL | 变动前状态 |
| after_status | INT | NOT NULL | 变动后状态 |
| operator_id | BIGINT UNSIGNED | NOT NULL | 操作人ID |
| operator_name | VARCHAR(100) | NOT NULL | 操作人姓名 |
| remark | VARCHAR(500) | NULL | 备注 |
| created_at | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 5.2 状态枚举

| 订单类型 | 状态值 | 状态名称 |
|---------|--------|---------|
| 采购订单 | 1 | 待审核 |
| 采购订单 | 2 | 已审核 |
| 采购订单 | 3 | 已入库 |
| 采购订单 | 4 | 已取消 |
| 销售订单 | 1 | 待审核 |
| 销售订单 | 2 | 已审核 |
| 销售订单 | 3 | 已出库 |
| 销售订单 | 4 | 已取消 |

## 6. 业务流程

### 6.1 采购订单状态变动流程

```
采购订单创建 → 状态=1(待审核)
       ↓
审核操作 → 记录日志(before=1, after=2) → 状态=2(已审核)
       ↓
入库操作 → 记录日志(before=2, after=3) → 状态=3(已入库)

取消操作 → 记录日志(before=1, after=4) → 状态=4(已取消)
```

### 6.2 销售订单状态变动流程

```
销售订单创建 → 状态=1(待审核)
       ↓
审核操作 → 记录日志(before=1, after=2) → 状态=2(已审核)
       ↓
出库操作 → 记录日志(before=2, after=3) → 状态=3(已出库)

取消操作 → 记录日志(before=1, after=4) → 状态=4(已取消)
```
