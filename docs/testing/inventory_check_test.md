# 库存盘点功能测试用例

## 1. 功能概述

库存盘点用于核对实际库存与系统库存的差异，确保账实相符。

### 1.1 核心业务流程

```
创建盘点单 → 录入盘点明细 → 提交盘点单 → 生成调整申请
```

### 1.2 状态定义

| 盘点单状态 | 值 | 说明 |
|-----------|-----|------|
| 进行中 | 1 | 初始状态，可修改盘点数据 |
| 已完成 | 2 | 提交后状态，不可修改 |

| 盘点明细状态 | 值 | 说明 |
|-------------|-----|------|
| 待处理 | 1 | 初始状态 |
| 已处理 | 2 | 提交后状态 |

## 2. 功能测试点

### 2.1 创建盘点单

| 用例编号 | 用例名称 | 测试步骤 | 预期结果 |
|---------|---------|---------|---------|
| IC-TC-001 | 正常创建盘点单 | 1. 选择有库存的仓库<br>2. 点击创建盘点单<br>3. 填写备注（可选）<br>4. 提交 | 1. 盘点单创建成功<br>2. 系统自动获取该仓库所有产品的系统库存<br>3. 明细初始值：actual_qty=0, diff_qty=-system_qty<br>4. 返回盘点单信息，包含明细列表 |
| IC-TC-002 | 空仓库创建盘点单 | 1. 选择无库存的仓库<br>2. 点击创建盘点单 | 1. 盘点单创建成功<br>2. 明细列表为空<br>3. 提示"该仓库无库存产品" |
| IC-TC-003 | 不存在的仓库 | 1. 选择不存在的仓库ID<br>2. 点击创建盘点单 | 返回错误：仓库不存在 |
| IC-TC-004 | 盘点单号格式 | 1. 创建盘点单<br>2. 检查单号格式 | 单号格式为 CK-YYYYMMDD-XXXX（如 CK-20260506-0001） |

### 2.2 盘点明细录入

| 用例编号 | 用例名称 | 测试步骤 | 预期结果 |
|---------|---------|---------|---------|
| IC-TC-005 | 正常录入盘点数量 | 1. 打开盘点单详情<br>2. 录入实际盘点数量<br>3. 保存 | 1. 差异数量自动计算：diff_qty = actual_qty - system_qty<br>2. 盘点单总差异自动更新 |
| IC-TC-006 | 录入负数数量 | 1. 录入实际盘点数量为负数<br>2. 保存 | 返回错误：实际数量不能为负数 |
| IC-TC-007 | 修改已录入数据 | 1. 已录入盘点数量<br>2. 修改实际数量<br>3. 保存 | 差异数量正确重新计算 |
| IC-TC-008 | 修改已提交单据 | 1. 盘点单状态为已完成<br>2. 尝试修改盘点数量 | 返回错误：只能修改进行中的盘点单 |

### 2.3 提交盘点单

| 用例编号 | 用例名称 | 测试步骤 | 预期结果 |
|---------|---------|---------|---------|
| IC-TC-009 | 正常提交盘点单 | 1. 完成所有盘点明细录入<br>2. 点击提交 | 1. 盘点单状态变为已完成<br>2. 所有明细状态变为已处理<br>3. 计算并更新总差异数量<br>4. 不能再修改盘点数据 |
| IC-TC-010 | 重复提交已提交单据 | 1. 盘点单状态为已完成<br>2. 再次点击提交 | 返回错误：只能提交进行中的盘点单 |
| IC-TC-011 | 提交不存在的盘点单 | 1. 提交一个不存在的盘点单ID | 返回错误：盘点单不存在 |
| IC-TC-012 | 提交后检查总差异 | 1. 录入多个盘点明细<br>2. 提交<br>3. 检查总差异 | total_diff = SUM(diff_qty) |

### 2.4 生成调整申请

| 用例编号 | 用例名称 | 测试步骤 | 预期结果 |
|---------|---------|---------|---------|
| IC-TC-013 | 生成盘盈调整申请 | 1. 盘点明细中存在 actual_qty > system_qty<br>2. 点击生成调整申请 | 1. 生成调整类型为"盘盈"的申请<br>2. adjust_type = 1<br>3. 调整数量 = diff_qty（正数） |
| IC-TC-014 | 生成盘亏调整申请 | 1. 盘点明细中存在 actual_qty < system_qty<br>2. 点击生成调整申请 | 1. 生成调整类型为"盘亏"的申请<br>2. adjust_type = 2<br>3. 调整数量 = abs(diff_qty) |
| IC-TC-015 | 生成无差异调整申请 | 1. 所有盘点明细 diff_qty = 0<br>2. 点击生成调整申请 | 返回：成功生成0个库存调整申请 |
| IC-TC-016 | 对进行中单据生成调整 | 1. 盘点单状态为进行中<br>2. 点击生成调整申请 | 返回错误：只能对已完成的盘点单生成调整单 |
| IC-TC-017 | 调整申请关联盘点单 | 1. 生成调整申请<br>2. 查看调整申请详情 | 1. source_type = 2（盘点）<br>2. source_id = 盘点单ID<br>3. reason 包含盘点单号 |

## 3. 边界测试用例

| 用例编号 | 测试场景 | 输入 | 预期结果 |

  "warehouseId": 1,

}

    "items": [
      {
        "id": 1,

        "productId": 1,

## 5. 数据库验证

### 5.1 inventory_checks 表

| 字段 | 类型 | 说明 | 验证方法 |
|------|------|------|---------|
| id | BIGINT UNSIGNED | 主键 | 自动递增 |
| check_no | VARCHAR(50) | 盘点单号 | 格式：CK-YYYYMMDD-XXXX |
| warehouse_id | BIGINT UNSIGNED | 仓库ID | 关联 warehouses 表 |
| status | INT | 状态 | 1:进行中, 2:已完成 |
| total_diff | INT | 总差异 | = SUM(diff_qty) |
| remark | VARCHAR(500) | 备注 | 可为空 |

### 5.2 inventory_check_items 表

| 字段 | 类型 | 说明 | 验证方法 |
|------|------|------|---------|
| id | BIGINT UNSIGNED | 主键 | 自动递增 |
| check_id | BIGINT UNSIGNED | 盘点单ID | 关联 inventory_checks 表 |
| product_id | BIGINT UNSIGNED | 产品ID | 关联 products 表 |
| system_qty | INT | 系统库存 | 创建时从 warehouse_inventories 获取 |
| actual_qty | INT | 实际盘点 | 用户录入，初始为0 |
| diff_qty | INT | 差异数量 | = actual_qty - system_qty |
| status | INT | 状态 | 1:待处理, 2:已处理 |

## 6. 测试数据准备

### 6.1 前提条件

1. 存在至少一个仓库
2. 仓库中有产品库存记录（warehouse_inventories 表）

```sql
-- 1. 创建测试仓库

INSERT INTO warehouse_inventories (product_id, warehouse_id, quantity) VALUES (1, 1, 100);
```

## 8. 风险评估

| 风险点 | 影响 | 建议 |
|--------|------|------|
| 并发提交同一盘点单 | 数据不一致 | 添加分布式锁 |
| 盘点过程中库存变动 | 盘点数据不准确 | 盘点期间锁定库存操作 |
| 大量产品时性能 | 创建盘点单耗时过长 | 分批处理，添加进度提示 |
