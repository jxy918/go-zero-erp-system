# 库存调拨功能测试方案

## 1. 测试概述

### 1.1 测试目标
- 验证库存调拨功能的完整性和正确性
- 确保数据库操作的安全性和准确性
- 确保前后端交互的正常运行
- 验证业务流程的正确性

### 1.2 测试范围
- 数据库表结构和数据迁移
- 后端API接口功能
- 前端页面操作
- 业务流程（创建-审核-执行）
- 权限控制

## 2. 测试环境

### 2.1 环境配置
- 数据库：MySQL 5.7+
- 后端：Go 1.20+
- 前端：Node.js 16+
- 浏览器：Chrome/Firefox/Edge

### 2.2 测试数据准备
- 测试仓库：仓库A、仓库B
- 测试产品：产品A（库存：仓库A 100个
- 测试用户：管理员、仓库主管、仓库管理员

| API-015 | 执行调拨-状态不正确 | 执行非已审核状态的调拨单 | 执行失败 |
