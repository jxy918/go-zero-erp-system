# 订单状态变动日志功能测试文档

## 1. 测试概述

### 1.1 测试目的
验证订单状态变动日志功能的正确性和完整性，确保采购订单和销售订单的状态变动都能正确记录到日志表中。

### 1.2 测试范围
- 采购订单状态变动日志记录
- 销售订单状态变动日志记录
- 订单日志查询接口

### 1.3 测试环境
- 后端服务: http://localhost:8001
- 前端服务: http://localhost:3000
- 数据库: MySQL 8.0

## 3. 测试步骤

### 3.1 采购订单审核日志测试
1. 登录系统（admin/admin123）
2. 进入采购管理 → 采购订单列表
3. 创建新采购订单（选择供应商、仓库、产品）
4. 点击审核按钮
5. 查看数据库order_logs表，确认有新记录

### 3.2 销售订单审核日志测试
1. 登录系统（admin/admin123）
2. 进入销售管理 → 销售订单列表
3. 创建新销售订单（选择客户、仓库、产品）
4. 点击审核按钮
5. 查看数据库order_logs表，确认有新记录

### 3.3 日志查询接口测试
1. 调用 GET /order-log/list
2. 验证返回格式正确
3. 验证数据完整性
